The API Package contains a general class (Utilities) and specific classes, External, Librarian, etc. Organization:

file Xyz.py contains class Xyz.py, only one class per file.
some files, e.g., Utilities, External, create a singleton.  The singleton created by class Xyz.py is named xyz.
some classes are multi-hierarchical, e.g., Utilities ( Files, Texts, ... )
some classes have a configuration file, e.g., Xyz.py -> xyz_configuration.py. it is embedded directly in the class (from ..._configuration import *)
files in lower case are scripts, e.g., to launch a class.



To use the package (interactive version. You can do the same from a script)

go to parent directory of library (xxx/library/ -> xxx/),
run python (global installation or local copy library/python/python )
import library (configures the sys.path, os.environ variables)
from guiPygame.Xyz import *

# here, singleton xyz is created)


General purpose class:

  • Utilities (superclasses Texts, PathNames, Files, Classes, Items) : wraps string and list methods (to capture exceptions),  path naming (names, extensions, etc.). Provides advanced functions for text processing (e.g., searches) files and directories (e.g., instantiation).classes used specifically in FTK Doc
Useful classes:
  • Archiver: backups in zip format.
  • AdhocNetwork, BasicNetwork, NetworkNode: ad-hoc network using udp and tcp (designed for domestic WIFI).
  • BasicWebServer, HttpHandler. simple HTTP server
  • Benchmark. CPU benchmark on typical Python operations
  • Clock: time and date functions, timers.
  • Documentation (all files documentation*.* (can be used standalone) : on-line help and documentation (files in diffent formats: html, ascii, tab-separated).
  • External : access to external files. Wraps OS and external processes commands), overrides file extension associations of the OS.
  • Logger : logs in tab-separated format.
  • Setup : functions to install and deinstall a software
  • Sound : functions to play sound files
  • TableFile, TableData, AttributeValueList: formatted files IO (tab-separated, read/writes matrices or lists..)
Specific of documentation application FTK Doc
  • Book : manages directories of notes (laboratory book, pending notes..)
  • Context (superclasses Paths, Messages, Rights, Types, Variables, uses TypeNode) : context of execution. a class (and a singleton) containing a set of standard paths, text messages, access rights, predefined types of documents (with templates), predefined access rights, context variables (e.g., to instantiate in texts).
  • Converter, BibFile, Html, PdfFile, Wiki, XmlFile: file converter (ascii, wiki, html), and read/write functions for specific formats
  • Librarian: manages a bookcase, i.e., a volume used as document repository.
Sandbox

  • Serial, I2C
  • Statistics
Others

  • compile,py, findLibrary.py...: should not be used.
  • documentation*.html, *.txt, *.tsv: templates for documentation.
