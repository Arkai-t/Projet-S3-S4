To prepare an application that uses the library.

1 - place in one directory (we call it  root) :
configuration : configuration files. you can place copy of whatever xxx_configuration.py file of the library here. Plus your own files.
library
data :  input and output data
program :  your program
execute.bat (copy it from library/bat) and/or python.bat (same script, if you want to to fool the command line)

2- prepare 3 files to configure the library

configure the search paths of your applications - library/sysPath.txt

create file library/sysPath.txt with the line ../program/ (and whatever search path you want. use absolute paths or paths relative to root/library/

are you standalone?- library/standalone.txt

if your application DO NOT require anything else, create file library/standalone.txt. This means that the search paths will be limited to root/, root/library/ and subdirectories.

do you want to launch automatically a file?- library/launch.txt

if you want to execute automatically some Python files every time the library is loaded, create a file library/launch.txt  and place one file per line. Use absolute paths or paths relatice to root/library


To execute an application that uses the library

Prepare the application and the library as seen above. Here are the ways to execute your application

directly from the OS. Run execute.bat (or python.bat ) from directory root/. The scripts in  library/launch.txt will be executed.

from the console. In root/, type execute . It is the same as above.

from the console. In root/, type execute my_script.py. This prepares sys.path and os.environ and  launches your script. Overrides library/launch.txt.

interactively. From the console, in root/ type library\python\python (local copy).

from the python shell, type import library, then whatever is necessary to run your application.
